# 用户流程

## 1. 首次启动流程

1. 用户打开 App。
2. 看到一句核心说明：实时看看今天赚了多少钱。
3. 选择工资模式：月薪、日薪、时薪。
4. 输入工资金额。
5. 设置上班时间和下班时间。
6. 选择每周工作日。
7. 可选设置午休时间。
8. 进入首页。

成功结果：

- 首页显示今日已赚金额。
- 首页显示距离下班时间。
- 用户知道 App 已经可用。

## 2. 日常查看流程

1. 用户打开 App 或查看小组件。
2. 看到今日已赚金额。
3. 看到距离下班时间。
4. 如果感兴趣，进入消费换算。

关键要求：

- 打开 App 不应要求用户再次确认。
- 首页必须直接显示核心数据。

## 3. 消费换算流程

1. 用户点击首页的消费换算入口。
2. 输入消费金额。
3. App 展示需要工作多久。
4. 用户可选择保存记录。

示例：

- 用户输入 35。
- App 展示：这笔消费需要你工作 42 分钟。

## 4. 添加小组件流程

1. 用户在首页看到添加小组件提示。
2. App 展示小组件预览图。
3. 用户去系统桌面添加小组件。
4. 小组件展示今日已赚金额或下班倒计时。

注意：

- App 不能直接替用户添加系统小组件，只能提供引导。
- 引导文案要短，不做冗长教程。

## 5. Pro 转化流程

触发场景：

- 用户选择高级小组件样式。
- 用户选择高级主题。
- 用户添加第二份工作配置。
- 用户查看完整统计。

流程：

1. 用户点击 Pro 功能。
2. 展示 Pro 权益页。
3. 用户选择购买。
4. 支付成功后立即解锁。

Pro 页面重点：

- 先展示用户想要的功能。
- 再展示全部权益。
- 价格和恢复购买入口必须清晰。

## 6. 修改设置流程

1. 用户进入设置页。
2. 修改工资、工作日或上下班时间。
3. 保存后回到首页。
4. 首页按新规则重新计算。
5. 小组件读取新配置。

要求：

- 修改设置不删除历史换算记录。
- 如果当天计算规则变化，首页使用最新设置重新计算当天状态。
