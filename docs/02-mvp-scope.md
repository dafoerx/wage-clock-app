# MVP 范围

## 1. MVP 目标

用最小成本验证：

- 用户是否愿意设置工资和工作时间。
- 用户是否愿意每天查看今日已赚金额。
- 用户是否会添加桌面小组件。
- 用户是否愿意为主题、小组件、多工作或统计付费。

## 2. 首版必须做

### 设置

- 月薪、日薪、时薪三种工资模式。
- 工作日选择。
- 上班和下班时间。
- 午休开关和时间段。
- 本地保存设置。

### 首页

- 今日已赚金额。
- 距离下班倒计时。
- 工作进度条。
- 今日工作状态。
- 今日剩余可赚金额。

### 消费换算

- 输入金额。
- 输出需要工作多久。
- 常用金额快捷按钮。

### 小组件

- 小号小组件：今日已赚金额。
- 中号小组件：金额、倒计时、进度。

### 付费

- Pro 展示页。
- StoreKit 商品配置预留。
- 至少 2 个付费触发点：主题、小组件样式。

## 3. 首版不做

- 登录注册。
- 云同步。
- 银行账户或支付平台同步。
- 自动导入工资单。
- 复杂记账。
- 社区分享。
- AI 预算建议。
- 企业排班。
- Android 版本。

## 4. 验收标准

功能验收：

- 新用户能在 60 秒内完成首次设置。
- 工作中状态下，今日已赚金额随时间增长。
- 午休时间内，收入暂停增长。
- 休息日显示休息状态，不累计收入。
- 已下班后，今日已赚金额不再增长，除非开启加班功能。
- 消费换算结果正确。
- 小组件能读取当前工作配置。

体验验收：

- 首页无需解释即可理解。
- 核心数字足够醒目。
- 消费换算输入路径不超过 2 次点击。
- 免费版没有强制登录。

商业验收：

- 用户点击高级主题时出现 Pro 页面。
- 用户点击高级小组件时出现 Pro 页面。
- Pro 页面清楚展示权益和价格。

## 5. MVP 发布建议

第一版可以先做一次性买断，等产品验证后再加入订阅。

原因：

- 工具 App 用户对低价买断更容易接受。
- 初创阶段先验证支付意愿，不必过早优化订阅模型。
- 订阅需要持续价值，首版内容还不够厚。
