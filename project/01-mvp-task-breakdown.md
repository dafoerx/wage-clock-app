# MVP 开发任务拆分

这些条目可以直接转成 GitHub Issues。

## Milestone 1：工程与基础模型

### Issue 1：创建 iOS SwiftUI 工程

目标：

- 建立主 App 工程。
- 设置 bundle id。
- 建立基础目录结构。

验收：

- App 可启动。
- 首页显示占位内容。

### Issue 2：实现数据模型

目标：

- 定义 `WorkProfile`。
- 定义 `ExpenseRecord`。
- 定义 `UserSettings`。

验收：

- 模型可 Codable。
- 有基础示例数据。

### Issue 3：实现本地存储

目标：

- 用 UserDefaults 保存当前工作配置。
- 支持读取和更新。

验收：

- 关闭 App 后配置不丢失。

## Milestone 2：计算核心

### Issue 4：实现工资计算服务

目标：

- 支持月薪、日薪、时薪。
- 支持工作日。
- 支持午休扣除。

验收：

- 单元测试覆盖主要计算场景。

### Issue 5：实现消费换算服务

目标：

- 输入金额，输出需要工作分钟。
- 格式化为分钟、小时、天。

验收：

- 不同金额和工资模式下计算正确。

## Milestone 3：核心页面

### Issue 6：首次设置流程

目标：

- 工资模式选择。
- 工资金额输入。
- 工作日选择。
- 上下班时间选择。

验收：

- 新用户可以完成设置并进入首页。

### Issue 7：首页

目标：

- 展示今日已赚。
- 展示下班倒计时。
- 展示进度条。
- 展示工作状态。

验收：

- 前台每秒刷新。
- 非工作日显示休息。

### Issue 8：消费换算页

目标：

- 输入消费金额。
- 展示换算结果。
- 常用消费快捷按钮。

验收：

- 输入后立即展示结果。

### Issue 9：设置页

目标：

- 查看和编辑工资设置。
- 查看 Pro 入口。
- 查看关于信息。

验收：

- 修改设置后首页重新计算。
